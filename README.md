# ProtoBufferParser

Google Protocol Buffer IDL (.proto) 파일을 Unreal Engine C++ 구조체 (.h / .cpp) 로 변환하는 컴파일러입니다.

## 개요

- **입력**: `.proto` 파일 (Proto3)
- **출력**: Unreal Engine 호환 `.h` / `.cpp` 파일
- **파서**: ANTLR4 기반 렉서 + 투패스 파서
- **런타임**: .NET 8.0 (C# 12)

`protoc`이 생성한 C++ 코드를 전제로, 각 Unreal 구조체에 마샬링 생성자를 포함하여 Proto 타입에서 Unreal 타입으로 자동 변환합니다.

## 요구사항

- [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) 이상

## 빌드

```bash
dotnet build
```

## 사용법

```bash
dotnet run -- -i <입력 디렉토리> -o <출력 디렉토리> [-v]
```

| 옵션 | 설명 |
|---|---|
| `-i`, `--input-dir` | (필수) `.proto` 파일이 있는 입력 디렉토리 |
| `-o`, `--output-dir` | (필수) 생성된 `.h`/`.cpp` 파일을 저장할 출력 디렉토리 |
| `-v`, `--verbose` | 상세 로그 출력 활성화 |

### 예시

```bash
dotnet run -- -i ./samples -o ./output -v
```

## 네이밍 규칙

| 항목 | 규칙 | 예시 |
|---|---|---|
| 구조체 | `F{Name}Proto` | `FPlayerProto` |
| Enum | `E{Name}Proto` | `EItemTypeProto` |
| 파일명 | `ME{Name}Proto.h/.cpp` | `MEPlayerProto.h` |
| 중첩 메시지 | Flat 풀기 | `Outer.Inner` -> `FOuterInnerProto` |

## 지원 타입 매핑

| Proto3 타입 | Unreal C++ 타입 |
|---|---|
| `int32`, `sint32`, `sfixed32` | `int32` |
| `int64`, `sint64`, `sfixed64` | `int64` |
| `uint32`, `fixed32` | `uint32` |
| `uint64`, `fixed64` | `uint64` |
| `float` | `float` |
| `double` | `double` |
| `bool` | `bool` |
| `string` | `FString` |
| `bytes` | `TArray<uint8>` |
| `repeated T` | `TArray<T>` |
| `map<K, V>` | `TMap<K, V>` |
| message 참조 | `F{Name}Proto` |
| enum 참조 | `E{Name}Proto` |

## 입출력 예시

### 입력: `common.proto`

```protobuf
syntax = "proto3";

message Vector3 {
    float x = 1;
    float y = 2;
    float z = 3;
}
```

### 출력: `MEVector3Proto.h`

```cpp
// ----------------------------------------------
// AUTO-GENERATED from common
// DO NOT MODIFY - Changes will be overwritten
// Generated by ProtoBufferParser
// ----------------------------------------------
#pragma once

#include "CoreMinimal.h"
#include "MEVector3Proto.generated.h"

USTRUCT(BlueprintType)
struct FVector3Proto
{
    GENERATED_BODY()

    FVector3Proto() = default;

    // Marshal from protobuf message
    explicit FVector3Proto(const ::Vector3& proto);

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Proto")
    float X;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Proto")
    float Y;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Proto")
    float Z;
};
```

### 출력: `MEVector3Proto.cpp`

```cpp
// ----------------------------------------------
// AUTO-GENERATED from common
// DO NOT MODIFY - Changes will be overwritten
// Generated by ProtoBufferParser
// ----------------------------------------------
#include "MEVector3Proto.h"
#include "common.pb.h"

FVector3Proto::FVector3Proto(const ::Vector3& proto)
{
    X = proto.x();
    Y = proto.y();
    Z = proto.z();
}
```

## 파이프라인 구조

1. **파일 스캔**: 입력 디렉토리에서 `.proto` 파일 재귀 탐색
2. **의존성 분석**: regex로 `import` 문 추출 후 의존성 그래프 구축
3. **위상 정렬**: Kahn 알고리즘으로 파싱 순서 결정 (순환 의존성 검출)
4. **파싱**: ANTLR4 투패스 파서로 각 파일을 AST로 변환 (크로스파일 타입 참조 지원)
5. **평탄화**: 중첩 메시지/enum을 최상위 레벨로 풀기
6. **코드 생성**: AST에서 Unreal Engine C++ 코드 생성
7. **파일 출력**: `.h` / `.cpp` 파일 저장

## 프로젝트 구조

```
ProtoBufferParser/
├── Models/          # AST 노드 및 데이터 모델
├── Interfaces/      # 인터페이스 정의
├── Parsers/         # ANTLR4 렉서/파서 및 AST 빌더
├── Services/        # 핵심 비즈니스 로직 (코드 생성, 타입 매핑, 평탄화 등)
├── Utilities/       # 네이밍 헬퍼 유틸리티
├── Exceptions/      # 커스텀 예외 클래스
├── samples/         # 샘플 .proto 파일
├── output/          # 생성된 출력 파일
└── Program.cs       # CLI 엔트리포인트
```
