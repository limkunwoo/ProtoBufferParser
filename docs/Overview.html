ProtoBufferParser 문서 Overview

.proto 파일을 Unreal Engine C++ 구조체로 변환하는 컴파일러의 내부 동작을 설명하는 5개 문서의 종합 안내서입니다.

1. 프로젝트 소개

ProtoBufferParser는 Google Protocol Buffer IDL 파일(.proto)을 읽어 Unreal Engine 호환 C++ 구조체(.h / .cpp)를 생성하는 소스-to-소스 컴파일러입니다.

입력과 출력

항목내용
입력.proto 파일 (Protocol Buffer 3 IDL)
출력.h / .cpp 파일 (Unreal Engine C++ 구조체)
파서 기술ANTLR4 (렉서 + 파서 자동 생성)
언어C# 12 / .NET 8.0

네이밍 규칙

종류패턴예시
StructF{MessageName}ProtoFPlayerProto
EnumE{EnumName}ProtoEStatusProto
파일ME{MessageName}Proto.h/.cppMEPlayerProto.h
중첩F{Outer}{Inner}ProtoFPlayerInventoryProto

핵심 아이디어

이 프로젝트는 단순한 텍스트 변환이 아닙니다. .proto 파일의 문법 구조를 완전히 이해하고, 타입 간 관계(전방 참조, 중첩 메시지, 스코프)를 해석한 뒤, Unreal Engine의 리플렉션 시스템(USTRUCT, UPROPERTY)에 맞는 코드를 생성합니다.

이 과정은 크게 5단계 파이프라인으로 이루어집니다.

각 단계 요약

단계입력출력핵심 동작
1. 렉싱 소스 문자열 토큰 스트림 공백/주석 제거, 키워드·심볼·리터럴·식별자로 분류 → 상세
2. 파싱 토큰 스트림 CST (Parse Tree) 재귀 하강 파싱, 2-Pass로 전방 참조 해결, 시맨틱 프레디킷으로 타입 판별
3. CST → AST CST AST Visitor 패턴으로 순회, 구두점·래퍼 규칙 제거, 의미 정보만 추출 → 상세
4. 코드 생성 AST UE C++ 코드 타입 매핑 (string→FString, repeated→TArray), UE 매크로 삽입 → 상세

핵심 특징: 2-Pass 파싱

Proto3는 아직 선언되지 않은 타입을 참조할 수 있습니다 (전방 참조). 이를 해결하기 위해 파서는 2번 토큰 스트림을 순회합니다:

  • Pass 1 — 모든 message와 enum 이름을 심볼 테이블에 수집 (파싱 결과는 버림)
  • Pass 2 — 심볼 테이블을 참조하여 정확한 CST 생성

핵심 특징: 노드 압축

CST는 문법 규칙을 그대로 반영하므로 구두점(=, ;, {)과 래퍼 규칙(fieldName → ident → IDENTIFIER) 같은 불필요한 노드가 많습니다. AST 변환 시 이들을 제거하여 ~29개 CST 노드 → ~3개 AST 노드로 극적으로 압축합니다.

3. 문서 가이드

아래 5개 문서가 파이프라인의 전 과정을 커버합니다. 각 문서는 독립적으로 읽을 수 있지만, 순서대로 읽으면 전체 흐름이 자연스럽게 연결됩니다.

ANTLR4 g4 문법 파일: 구조와 작성법 Q&A 8섹션

파이프라인 범위: 문법 정의 — Lexer와 Parser가 따르는 규칙 자체를 이해합니다.

  • 형식 문법(Formal Grammar)과 EBNF의 개념
  • EBNF 핵심 연산자 (|, ?, *, +)
  • .g4 파일 구조: grammar 선언, Parser Rules, Lexer Rules
  • Parser Rule vs Lexer Rule의 근본적 차이
  • 반복 패턴: 쉼표 리스트, 블록 감싸기, 재귀, 래퍼 규칙
  • Protobuf3.g4 규칙 계층 분석 (proto → messageDef → field)
  • 시맨틱 프레디킷과 액션
  • 문법 작성 팁: 좌재귀, 렉서 우선순위, 최장 매치, skip vs channel(HIDDEN)

Protobuf3.g4 문법 파일 상세 설명 레퍼런스 26섹션

파이프라인 범위: 문법 레퍼런스 — Protobuf3.g4의 모든 규칙을 한 줄씩 해설합니다.

  • 헤더/옵션, 2-Pass 진입점, 최상위 구조
  • syntax, import, package, option 선언
  • 필드 레이블, oneof, map, 타입 참조
  • message, enum, extend, service 정의
  • Lexer 규칙: 키워드 27개, 심볼 15개, 리터럴, 공백/주석
  • 시맨틱 액션 전용 규칙 (doEnterBlock, doMessageNameDef 등)
  • 접이식 원본 전문 (685줄) 포함

토큰 스트림 → CST 구축: 렉싱에서 Parse Tree까지 Q&A 7섹션

파이프라인 범위: 1단계 렉싱 + 2단계 파싱 — 소스 텍스트가 CST가 되는 전 과정입니다.

  • 렉싱 파이프라인: AntlrInputStream → Lexer → CommonTokenStream
  • 토큰 분류: 키워드 27개, 심볼 15개, 리터럴 5개, 히든 채널
  • 재귀 하강 파싱: EnterRule → Match → ExitRule
  • Context 클래스: 단수/복수 접근자, TerminalNode vs Context
  • 2-Pass 파싱과 DoRewind() 내부 동작
  • 시맨틱 프레디킷: IsMessageType_(), IsEnumType_()
  • 단계별 추적: 14개 토큰 → 19개 CST 노드 (인터랙티브 스테퍼)

CST → AST 변환: Parse Tree에서 Abstract Syntax Tree까지 Q&A 7섹션

파이프라인 범위: 3단계 AST 변환 — CST에서 의미 있는 정보만 추출하여 AST를 생성합니다.

  • 파싱 파이프라인 전체 흐름 (5단계 조감도)
  • CST vs AST: 구조적 차이와 노드 압축 (29 → 3)
  • Visitor 패턴: AstBuilder, VisitField, VisitMessageDef
  • Visitor vs Listener 비교
  • 변환 과정 단계별 추적 (인터랙티브 스테퍼)
  • 타입 판별과 2-Pass 파싱의 역할
  • 코드 생성 개요: AST → UE C++ (상세 문서)

AST → Unreal C++ 코드 생성: 추상 구문 트리에서 USTRUCT/UENUM까지 Q&A 8섹션

파이프라인 범위: 4단계 코드 생성 — AST 노드를 Unreal Engine C++ 코드(.h/.cpp)로 변환하는 전 과정입니다.

  • 코드 생성 파이프라인 5단계 구조 (Graphviz 다이어그램)
  • MessageFlattener: 중첩 타입 평탄화 (FullName vs ProtocTypeName)
  • UnrealTypeMapper: proto3 → Unreal 타입 변환 테이블 (15개 스칼라 + 컨테이너)
  • EnumTemplate: UENUM 헤더 생성
  • StructTemplate: USTRUCT 헤더 + 마샬링 생성자 생성
  • 마샬링 8가지 패턴 (인터랙티브 스테퍼): string, primitive, bytes, repeated, map, optional, enum, message
  • 의존성 해결: DependencyGraph (Kahn 위상 정렬) + #include 체인
  • 전체 입출력 예시 추적: .proto → .h/.cpp 완전 변환

4. 읽는 순서 권장

파이프라인의 흐름을 따라가는 순서로 읽으면 가장 자연스럽습니다:

1
g4_EBNF_설명 — 먼저 ANTLR4 문법 파일이 무엇인지, 어떻게 읽는지를 이해합니다. EBNF 연산자, Parser Rule과 Lexer Rule의 차이, 핵심 패턴을 익힙니다.
2
Protobuf3_g4_설명 — 실제 Protobuf3.g4 파일의 모든 규칙을 한 줄씩 확인합니다. 1번 문서에서 배운 개념이 실전에서 어떻게 쓰이는지 볼 수 있습니다. 레퍼런스 용도로 필요할 때마다 돌아와서 참고하세요.
3
TokenStream_to_CST_설명 — 소스 텍스트가 토큰으로 분해되고, 파서가 토큰을 하나씩 소비하면서 CST를 조립하는 과정을 추적합니다. 2-Pass 파싱과 시맨틱 프레디킷의 실제 동작을 이해합니다.
4
CST_to_AST_설명 — CST에서 불필요한 노드를 제거하고 AST를 만드는 Visitor 패턴을 이해합니다. AST가 완성되면 코드 생성의 입력이 됩니다.
5
AST_to_Unreal_설명 — AST에서 최종 Unreal Engine C++ 코드를 생성하는 전 과정. 중첩 타입 평탄화, 8가지 마샬링 패턴, 의존성 위상 정렬까지. 파이프라인의 완성입니다.

각 문서는 독립적으로도 읽을 수 있습니다. 특정 단계만 궁금하다면 해당 문서로 바로 이동해도 됩니다.
